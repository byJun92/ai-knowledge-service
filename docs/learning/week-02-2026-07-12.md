## 5.1 pydantic-settings

### 무엇인가

`pydantic-settings`는 환경변수, `.env`, secrets 파일 등의 값을 타입이 지정된 설정 객체로 변환한다.

`BaseSettings`를 상속하면 다음을 수행한다.

- 환경변수 읽기
- 문자열을 `int`, `bool`, `Literal` 등으로 변환
- 필수값 누락 검증
- 허용되지 않은 값 검증
- 기본값 적용
- 테스트에서 생성자 값으로 덮어쓰기

### 프로젝트에서 담당하는 역할

```text
macOS 환경변수 / .env
          ↓
pydantic-settings
          ↓
Settings 객체
          ↓
FastAPI / DB / Collector / RAG / AgentRuntime
```

향후 다음 설정이 계속 추가된다.

- 데이터베이스 주소
- Vector DB 주소
- Embedding 모델명
- LLM 모델명
- API Key
- timeout
- retry
- token budget
- Agent iteration limit

### 설치

```bash
uv add pydantic-settings
```

변경되는 파일:

```text
pyproject.toml
uv.lock
.venv/
```

### 핵심 개념

- `BaseSettings`: 환경변수를 읽는 설정 모델
- `SettingsConfigDict`: 환경변수 prefix와 `.env` 파일 설정
- `SecretStr`: 출력 시 값을 마스킹하는 문자열 타입
- `Literal`: 허용 가능한 문자열 값 제한

### 꼭 기억할 명령어


## 5.2 structlog

### 무엇인가

`structlog`는 문자열 중심 로그가 아니라 key-value 기반 이벤트 로그를 만들기 위한 라이브러리다.

일반 문자열 로그:

```text
문서 처리 성공 doc-001 153ms
```

구조화 로그:

```json
{
  "event": "document_processed",
  "document_id": "doc-001",
  "latency_ms": 153,
  "level": "info",
  "timestamp": "2026-07-12T01:00:00Z"
}
```

### 왜 필요한가

향후 다음 값을 검색하고 집계해야 한다.

- `run_id`
- `agent_id`
- `document_id`
- `model`
- `latency_ms`
- `token_usage`
- `cost`
- `retry_count`
- `error_code`
- `stop_reason`

### Java/Spring 비교

| Java | Python |
|---|---|
| SLF4J | 표준 `logging` |
| Logback JSON Encoder | `structlog` JSON renderer |
| MDC | `structlog.contextvars` |
| key-value logging | event field |

### 설치

```bash
uv add structlog
```

### 이번 주 사용 범위

- local: 읽기 쉬운 console renderer
- production: JSON renderer
- timestamp
- log level
- logger name
- exception 정보
- key-value event

### 권장 형태

```python
logger.info(
    "document_processed",
    document_id="doc-001",
    latency_ms=153,
)
```

모든 값을 문자열로 합치지 않는다.

## 5.3 mypy

### 무엇인가

`mypy`는 Python 코드를 실행하지 않고 타입 힌트를 분석하는 정적 타입 검사기다.

```python
def add(left: int, right: int) -> int:
    return left + right


add("1", 2)
```

`mypy`는 실행 전에 잘못된 인자 타입을 찾는다.

### 설치와 실행

```bash
uv add --dev mypy
uv run mypy src tests
```

### strict 설정

```toml
[tool.mypy]
python_version = "3.13"
strict = true
mypy_path = "src"
packages = ["ai_knowledge_service"]
warn_unused_configs = true
pretty = true
show_error_codes = true
```

### 대표 대안

| 도구 | 특징 |
|---|---|
| mypy | 생태계와 학습 자료가 넓음 |
| Pyright | 빠른 검사와 VS Code/Pylance 연계 |
| Pyrefly | 새로운 고성능 타입 검사기 |
| ty | Astral이 개발 중인 타입 검사기 |

이번 프로젝트에서는 안정적인 학습 자료와 생태계를 우선해 `mypy`를 사용한다.

### 흔한 실수

1. `# type: ignore` 남발
2. `Any`로 모든 타입 문제 회피
3. `None` 가능성 무시
4. 외부 입력을 타입 힌트만 믿고 검증하지 않음
5. 코드 일부만 검사

## 7.2 Document 타입 개선

파일:

```text
src/ai_knowledge_service/domain/document.py
```

```python
from dataclasses import dataclass
from datetime import UTC, datetime
from enum import StrEnum
from typing import Self

from ai_knowledge_service.types import DocumentId


class DocumentType(StrEnum):
    PAPER = "paper"
    TECH_ARTICLE = "tech_article"
    DOCUMENTATION = "documentation"


@dataclass(frozen=True, slots=True)
class Document:
    document_id: DocumentId
    title: str
    source_url: str
    document_type: DocumentType
    collected_at: datetime

    @classmethod
    def create(
        cls,
        *,
        document_id: str,
        title: str,
        source_url: str,
        document_type: DocumentType,
    ) -> Self:
        normalized_id = document_id.strip()
        normalized_title = title.strip()
        normalized_url = source_url.strip()

        if not normalized_id:
            raise ValueError("document_id must not be blank")
        if not normalized_title:
            raise ValueError("title must not be blank")
        if not normalized_url.startswith(("https://", "http://")):
            raise ValueError("source_url must be an HTTP(S) URL")

        return cls(
            document_id=DocumentId(normalized_id),
            title=normalized_title,
            source_url=normalized_url,
            document_type=document_type,
            collected_at=datetime.now(UTC),
        )
```

새 개념:

- `Self`: 현재 클래스 자신을 반환한다는 의미
- `DocumentId`: 일반 문자열에 도메인 의미를 부여

## 7.3 설정 모델

파일:

```text
src/ai_knowledge_service/config.py
```

```python
from functools import lru_cache
from typing import Literal

from pydantic import SecretStr, field_validator
from pydantic_settings import BaseSettings, SettingsConfigDict


Environment = Literal["local", "test", "production"]
LogLevel = Literal["DEBUG", "INFO", "WARNING", "ERROR", "CRITICAL"]
LogFormat = Literal["console", "json"]


class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
        env_prefix="AIKS_",
        case_sensitive=False,
        extra="ignore",
    )

    app_name: str = "ai-knowledge-service"
    environment: Environment = "local"
    log_level: LogLevel = "INFO"
    log_format: LogFormat = "console"
    external_api_key: SecretStr | None = None

    @field_validator("app_name")
    @classmethod
    def validate_app_name(cls, value: str) -> str:
        normalized = value.strip()

        if not normalized:
            raise ValueError("app_name must not be blank")

        return normalized


@lru_cache
def get_settings() -> Settings:
    return Settings()


def clear_settings_cache() -> None:
    get_settings.cache_clear()
```

설계 설명:

- `env_prefix="AIKS_"`: 다른 서비스의 환경변수와 충돌 방지
- `lru_cache`: singleton 설정 객체처럼 재사용
- `SecretStr`: 출력 시 secret 마스킹
- `Literal`: 실행 환경과 로그 형식의 허용값 제한

## 7.5 애플리케이션 예외 계층

파일:

```text

```

```python
from collections.abc import Mapping
from typing import Any


class ApplicationError(Exception):
    code = "application_error"

    def __init__(
        self,
        message: str,
        *,
        details: Mapping[str, Any] | None = None,
    ) -> None:
        super().__init__(message)
        self.message = message
        self.details = dict(details or {})

    def to_dict(self) -> dict[str, object]:
        return {
            "code": self.code,
            "message": self.message,
            "details": self.details,
        }


class InvalidDocumentError(ApplicationError):
    code = "invalid_document"


class DocumentNotFoundError(ApplicationError):
    code = "document_not_found"


class ExternalServiceError(ApplicationError):
    code = "external_service_error"
```

서비스가 커지면 다음 오류를 구분해야 한다.

```text

DB 실패
LLM timeout
Embedding 실패
Agent 반복 한도 초과
```

예외의 `code`는 이후 FastAPI 오류 응답과 구조화 로그에서 공통으로 사용한다.

## 7.6 구조화 로깅

파일:

```python
import logging
import sys

import structlog

from ai_knowledge_service.config import Settings


def configure_logging(settings: Settings) -> None:
    shared_processors: list[structlog.types.Processor] = [
        structlog.contextvars.merge_contextvars,
        structlog.stdlib.add_logger_name,
        structlog.stdlib.add_log_level,
        structlog.processors.TimeStamper(fmt="iso", utc=True),
        structlog.processors.StackInfoRenderer(),
        structlog.processors.format_exc_info,
    ]

    renderer: structlog.types.Processor

    if settings.log_format == "json":
        renderer = structlog.processors.JSONRenderer()
    else:
        renderer = structlog.dev.ConsoleRenderer()

    structlog.configure(
        processors=[
            *shared_processors,
            structlog.stdlib.ProcessorFormatter.wrap_for_formatter,
        ],
        wrapper_class=structlog.stdlib.BoundLogger,
        logger_factory=structlog.stdlib.LoggerFactory(),
        cache_logger_on_first_use=False,
    )

    formatter = structlog.stdlib.ProcessorFormatter(
        foreign_pre_chain=shared_processors,
        processors=[
            structlog.stdlib.ProcessorFormatter.remove_processors_meta,
            renderer,
        ],
    )

    handler = logging.StreamHandler(sys.stdout)
    handler.setFormatter(formatter)

    root_logger = logging.getLogger()
    root_logger.handlers.clear()
    root_logger.addHandler(handler)
    root_logger.setLevel(settings.log_level)


def get_logger(name: str) -> structlog.stdlib.BoundLogger:
    return structlog.get_logger(name)
```

설계 포인트:

- 표준 `logging`과 `structlog`를 함께 사용
- 외부 라이브러리 로그도 동일 formatter로 출력 가능
- local에서는 console
- production에서는 JSON
- 테스트에서는 stdout을 캡처해 JSON field 검증

민감정보 금지:

- API Key
- access token
- refresh token
- 비밀번호
- 주민등록번호
- 원문 개인정보
- 전체 프롬프트와 전체 응답

```text

```

```markdown
# ADR-0002: 타입 기반 설정, 예외 계층, 구조화 로그 도입

## 상태

Accepted

## 배경

서비스가 FastAPI, 데이터 수집, 외부 API, RAG, Agent Runtime으로 확장되면
환경별 설정과 오류·로그를 일관되게 관리해야 한다.

## 결정

- 설정은 pydantic-settings의 BaseSettings로 관리한다.
- 환경변수 prefix는 AIKS_를 사용한다.
- 실제 비밀값은 .env 또는 배포 환경의 secret에 저장한다.

- local 환경은 console, production 환경은 JSON 로그를 기본으로 한다.
- 정적 타입 검사는 mypy strict 모드를 사용한다.

## 결과

장점:

- 설정 누락과 잘못된 값을 실행 초기에 발견한다.
- 오류 코드와 응답 형식을 통일할 수 있다.
- 로그를 검색·집계하기 쉽다.
- Agent run과 Tool 호출을 추적할 기반이 생긴다.
- 타입 오류를 CI 전에 로컬에서 발견할 수 있다.

비용:

- 설정과 테스트 구조가 초기 단계부터 다소 복잡해진다.
- mypy strict 오류를 해결하는 학습 비용이 있다.
- 구조화 로그 field naming 규칙을 지속적으로 관리해야 한다.
```

# 18. 최신 AI 논문 브리핑

## AgentTrace: A Structured Logging Framework for Agent System Observability

- 저자: Adam AlSayyad, Kelvin Yuxiang Huang, Richik Pal
- arXiv 등록일: 2026-02-07
- 원문: https://arxiv.org/abs/2602.10133

### 선정 이유

향후 AI Agent는 여러 Tool 호출, 반복 실행, 상태 변경, 비용과 token 사용, 부분 실패, 복구와 재시도를 포함한다.

이런 시스템은 최종 오류 메시지만으로 동작 원인을 파악하기 어렵다.

### 핵심 아이디어

Agent 실행 중 구조화 로그를 수집해 다음 세 범주를 관찰한다.

- operational
- cognitive
- contextual

단순 시스템 오류뿐 아니라 Agent 실행 흐름과 상태·맥락을 함께 추적하려는 접근이다.

### 프로젝트 적용점

Week 02부터 다음 원칙을 적용한다.

```text
모든 중요한 동작은 event 이름을 가진다.
ID와 측정값은 별도 field로 기록한다.
민감정보는 기록하지 않는다.
향후 run_id와 agent_id를 추가할 수 있게 설계한다.
```

### 한계

- arXiv preprint이므로 후속 검증이 필요하다.
- 로그가 많아지면 저장비용과 개인정보 위험이 증가한다.
- 구조화 로그 자체가 올바른 행동을 보장하지는 않는다.
- 무엇을 기록하고 제외할지 정책 설계가 필요하다.

### 사실과 해석

사실:

- 논문은 Agent 관찰을 위해 구조화된 runtime telemetry를 제안한다.
